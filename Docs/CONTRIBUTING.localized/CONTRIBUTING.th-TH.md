# แนวทางการมีส่วนร่วม

## ก่อนที่คุณจะเริ่ม

- เว้นแต่จะระบุไว้เป็นอย่างอื่นใน README ของโปรเจกต์นี้ โปรดแน่ใจว่าคุณโคลนซอร์สโค้ดจาก branch `main` โดยใช้พารามิเตอร์ `--recurse` เพื่อดึง submodules ทั้งหมดที่ Collapse ต้องใช้ในการคอมไพล์

    ```git clone https://github.com/neon-nyan/Collapse --recurse```

- โปรดใช้ IDE และ SDK ที่รองรับตามที่ระบุไว้ด้านล่าง
- โปรดอ่านหัวข้อ "ข้อจำกัดสำหรับฟีเจอร์ใหม่" ด้านล่าง
- เมื่อส่ง Pull Request (PR) โปรดแน่ใจว่าส่งไปยัง branch เดียวกับที่คุณดึงโค้ดมาในครั้งแรก เว้นแต่จะระบุไว้ใน README หรือได้รับคำแนะนำจากผู้ดูแลโปรเจกต์
- ไม่จำเป็นต้องเขียน PR อย่างละเอียดมาก แต่โปรดระบุให้ชัดเจนว่าเพิ่ม/ลบอะไร และโค้ดทำหน้าที่อะไร

## การมีส่วนร่วมด้านการแปลภาษา

คุณสามารถช่วยเพิ่มหรือพิสูจน์อักษรการแปลสำหรับ [Collapse บน Transifex!](https://explore.transifex.com/collapse-launcher/collapse-mainapp/)  
หากต้องการเพิ่มภาษาใหม่ที่ยังไม่มีในโปรเจกต์ Crowdin โปรดสร้าง issue ใน GitHub หรือพูดคุยผ่าน Crowdin เราจะพยายามตอบกลับโดยเร็วที่สุด

## เครื่องมือที่จำเป็น

ด้านล่างคือรายการเครื่องมือที่จำเป็นสำหรับการมีส่วนร่วมในโปรเจกต์นี้:
1. **Visual Studio 2022 (ทุกรุ่น - เวอร์ชันล่าสุด)**
   - เลือกคอมโพเนนต์ .NET desktop development
2. **Windows SDK (10.0.26100.0 เท่านั้น)** ผ่าน Visual Studio Installer
3. .NET 9 SDK: [**(9.0.4 หรือใหม่กว่า)**](https://dotnet.microsoft.com/en-us/download/dotnet/9.0)

> **หมายเหตุ**:  
> โปรดใช้ Visual Studio เวอร์ชันล่าสุดเสมอเพื่อให้สามารถเปิดโปรเจกต์ได้

## ข้อจำกัดสำหรับฟีเจอร์ใหม่

แม้ซอฟต์แวร์นี้จะเป็นโอเพ่นซอร์สและไม่มีความเกี่ยวข้องกับ HoYoverse, Cognosphere หรือบริษัทในเครือใด ๆ แต่เรายังคงต้องปฏิบัติตามข้อกำหนดการให้บริการและจรรยาบรรณของพวกเขาในการพัฒนา Collapse ซึ่งหมายความว่ามีบางฟีเจอร์ที่เรา **จะไม่** พัฒนา หากมี issue หรือ PR ที่เพิ่มฟีเจอร์เหล่านี้ เราจะปิดทันที ตัวอย่างฟีเจอร์ที่ไม่อนุญาต ได้แก่ แต่ไม่จำกัดเพียง:
- ฟีเจอร์ใด ๆ ที่มีปฏิสัมพันธ์กับ miHoYo SDK และ/หรือ API รวมถึง endpoint การยืนยันตัวตนและการชำระเงิน
- ฟีเจอร์ที่ inject โดยตรงเข้าสู่โปรเซสของเกม (หรือ subprocess) เช่น DLL injection, การอ่าน/แก้ไขหน่วยความจำ ฯลฯ
- ฟีเจอร์ที่แก้ไขไฟล์เกมหรือทรัพยากรโดยตรงเพื่อให้ได้เปรียบในเกมอย่างไม่เป็นธรรม

### ตัวอย่างฟีเจอร์ที่เราสนับสนุนให้ส่ง PR:

- ทรัพยากรจากชุมชน
- เครื่องมือที่ HoYoverse รับรองอย่างเป็นทางการ
- ฯลฯ

### ตัวอย่างฟีเจอร์ที่เราไม่น่าจะพัฒนา:

- ตัวสลับบัญชี
- ตัวปลดล็อก FPS (ที่ละเมิดกฎข้างต้น)
- โปรแกรมโกงหรือช่องโหว่ใด ๆ
- ฯลฯ
